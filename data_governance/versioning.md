# Versioning and Change Management

Für nachhaltige Governance muss jede Änderung an Daten oder Code versioniert sein.

**Versionierung**

- Schemata werden versioniert (v1, v2, …)
- Spark-Jobs dokumentieren Änderungen in Git
- Konfigurationsdateien werden versioniert

**Change Management**

- Änderungen müssen dokumentiert werden (Commit-Konventionen)
- Tests bei jeder Schema- oder Pipeline-Änderung
- Review-Prozess vor Deployment
